Dashboard Mobile Responsive

❌ NO IMPLEMENTADA — Crítica para usuarios reales

El archivo dashboard.css tiene 300 líneas y cero media queries. El sidebar de 280px fijo + el grid de 3 columnas se verá completamente roto en cualquier celular. La mayoría de tus usuarios probablemente entrarán desde móvil.

❌ Problemas concretos sin responsive
  • ✗ aside { width: 280px } fijo — en móvil ocupa la mitad de la pantalla
  • ✗ Widgets en grid-template-columns: repeat(3, 1fr) — se apilan deformes
  • ✗ Layout display: flex del body — sidebar + main horizontales, ilegible en 375px
🔧

Solución mínima (1 bloque de CSS): Agregar al final de dashboard.css un @media(max-width: 768px) que oculte el sidebar (o lo convierta en hamburger), cambie el grid a 1 columna, y ajuste paddings. Unas 20–30 líneas.
